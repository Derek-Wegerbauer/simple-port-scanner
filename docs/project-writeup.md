# Simple Port Scanner - Project Write-Up

## Summary

This project implements a simple TCP port scanner that identifies open and closed/filtered ports on a target system. It demonstrates core cybersecurity fundamentals including networking concepts, socket communication, command-line argument parsing, and basic reconnaissance techniques.

The tool is intended for educational use and authorized security testing as part of a cybersecurity portfolio.

## What This Tool Does and Why It Exists

This tool scans a user-specified target by attempting TCP connections to a defined range of ports and recording their status. It provides visibility into which network services may be exposed on a system, helping identify potential attack surfaces.

The project exists to reinforce a foundational understanding of how port scanning works at a low level, rather than relying only on advanced tools. It also bridges both defensive and offensive security perspectives by supporting service validation, firewall verification, and introductory penetration testing workflows.

## 1. Problem Definition

### Problem Statement

Organizations and individuals often lack clear visibility into which network services are exposed on their systems. Without an easy way to verify open ports, misconfigurations such as unintended services, weak firewall rules, or improperly segmented networks can go unnoticed.

This creates unnecessary security risk and increases the system's attack surface.

### Target User

- Cybersecurity students and beginners learning network reconnaissance
- IT and security practitioners performing basic service exposure checks
- Home lab users validating network configurations and firewall rules

### Success Criteria

- The tool successfully scans a target host and identifies open and closed/filtered TCP ports
- Results are displayed in a clear and readable format
- The scan completes reliably without crashing or hanging on unresponsive ports
- The tool is usable in an educational or lab environment with proper authorization

## 2. Scope and Requirements

### In Scope

## 3. Assumptions and Trade-Offs

### Assumptions

## 5. Architecture and Design

### High-Level Architecture

The Simple Port Scanner follows a linear, command-line-driven architecture. A user provides a target host and port selection as input, the scanner processes each port sequentially, and results are displayed directly to the console.

The design emphasizes clarity, reliability, and ease of understanding over performance optimization.

### Components

- Input Handler - Parses and validates the target, port range, timeout, and display options
- Scanner Engine - Attempts TCP socket connections to each specified port
- Timeout Controller - Limits connection attempts to prevent hanging on unresponsive ports
- Result Processor - Interprets connection responses as open or closed/filtered
- Output Module - Displays scan results in a readable console format

### Data Flow

1. User inputs target and scan options via the command line
2. Input is validated and passed to the scanner engine
3. The scanner engine attempts TCP connections to each selected port
4. Connection results are processed and classified
5. Results are printed to the console

### Trust Boundaries

- User to Tool - Assumes authorized and intentional use
- Tool to Network - Interacts with external systems over the network
- Tool to Output - Scan results are trusted only for the duration of execution

## 6. Tooling and Setup

### Technology Stack

- Programming Language: Python

  - `argparse` for command-line argument parsing and validation
  - `socket` for network connections
  - `sys` for error handling and program exit control

### Why These Choices?

Python was chosen for its readability, widespread use in cybersecurity tooling, and strong standard library support for networking.

The built-in `socket` library allows direct interaction with TCP connections, making it useful for demonstrating how port scanning works at a low level without relying on third-party tools.

Using only standard libraries keeps the project lightweight, portable, and easy to reproduce in most environments.

### Setup Instructions

1. Install Python 3.x on the host system

3. Confirm that no external packages are required
4. Run the tool from the command line with an appropriate target and scan option

## 8. Core Implementation

### Key Features

- TCP port scanning using `socket.connect_ex()`
- Support for scanning a built-in list of common ports
- Support for scanning user-defined port ranges
- Hostname and IPv4 address target resolution
- Configurable socket timeout values
- Open-port-only display mode with `--open-only`
- Command-line argument validation and error handling
- Clear console-based scan reporting and summary output

### How It Works

The tool performs a TCP connect scan against a target host specified by hostname or IPv4 address. The target is first resolved to an IPv4 address using DNS resolution when necessary. A list of ports is generated from either the built-in common port list or a user-specified range.

For each port, the scanner attempts to establish a TCP connection using Python's `socket` library. If the connection succeeds, the port is reported as open. If the connection fails, times out, or encounters a socket error, the port is reported as closed/filtered.

Users may optionally filter the output to display only open ports using the `--open-only` command-line option.

### Algorithms and Rules

- Sequential iteration through a defined port list or range
- TCP connect scanning using `socket.connect_ex()`
- DNS hostname resolution before scanning
- Timeout-based decision logic to prevent hanging connections
- Optional filtering of closed/filtered-port output
- Input validation for port ranges and timeout values
- Exception handling for network and socket-related errors

### Error Handling

- Missing required command-line arguments are detected by `argparse`
- Invalid port range formats are detected and reported
- Invalid timeout values are rejected before scanning begins
- Hostname resolution failures are reported to the user
- Socket timeouts are handled gracefully without crashing
- Network and socket exceptions are caught and reported as closed/filtered ports
- The scan continues even when individual connection attempts fail

## 13. Failure Modes and Edge Cases

### Failure Modes

- Target host is unreachable due to network issues or incorrect addressing
- Target system silently drops packets, causing connection timeouts
- Excessive timeout values significantly slow scan completion
- Local system firewall blocks outbound scan attempts
- Script may terminate early due to unexpected socket errors

### Edge Cases

- Scanning a very small port range, such as a single port
- Scanning a port range with all ports closed/filtered
- Target resolves to multiple IP addresses through DNS
- Scanning localhost versus remote hosts
- Running the tool with missing or incorrect command-line arguments
- Services bound only to IPv6 interfaces may appear closed because the current scanner supports IPv4 only

### Performance Limits

- Sequential scanning limits performance on large port ranges
- Scan duration increases with higher timeout values
- The tool is designed for small to moderate port ranges suitable for learning and validation
